# CookGPT / Solo Kitchen

**CookGPT 是一个面向独居创作者、Vibe Coder、独立创业者和高压知识工作者的 AI 一人食厨房 Skill。**

在 AI 高速发展的时代，注意力是人的第一财富。我们每天都在写代码、做产品、处理信息、应对 FOMO，却常常把吃饭这件最基本的事交给外卖、零食和临时决定。

CookGPT 想解决的不是“今天吃什么”这么简单的问题，而是：如何让一个忙碌的人，在不消耗太多决策力的前提下，稳定吃到简洁、营养、好吃、可执行的一餐。

它像一个轻量的个人厨师助理：根据你的口味、厨具、库存、预算、忌口和健康目标，从可信菜谱来源中提取可执行方案，生成一人食烹饪计划、买菜清单、火候提示和库存更新建议。

## 它适合谁

CookGPT 默认面向一人食场景，尤其适合：

- 独自生活的 Vibe Coder、开发者、设计师和创作者
- 独立创业者、OPC、小团队创始人
- 不想每天被“吃什么”消耗注意力的人
- 想减少外卖、开始规律下厨的新手
- 有减脂、增肌、控糖、忌口等需求的人
- 偶尔需要从一人食扩展到两人餐、家庭餐或仪式感晚餐的人

它的目标不是把你变成职业厨师，而是让你在真实生活里更容易吃好一点。

## 核心功能

### 1. 信源驱动的菜谱生成

CookGPT 不凭空编菜谱。它会优先使用：

- 本地已验证菜谱缓存
- 你的 Obsidian 菜谱笔记
- 可信 YouTube 创作者
- 结构化菜谱页面
- 经过验证的主流食谱网站

每次生成方案时，它会基于有食材、有步骤、有来源的菜谱，再根据你的 profile 做一人食适配。

### 2. 可执行的烹饪操作手册

它不只是给你“做法 1、2、3”。

CookGPT 会把菜谱转成更适合实际下厨的操作计划，包括：

- 食材用量表
- 备菜顺序
- 最终烹饪时间线
- 火候控制
- 熟度判断
- 风险提示
- Mermaid 操作流程图
- 必要时生成多菜并行时间线

目标是：你可以一边看这份操作手册，一边真的把饭做出来。

### 3. 智能买菜清单

CookGPT 可以根据菜谱和库存生成买菜清单，区分：

- 家里已有的食材
- 需要购买的食材
- 常备调料
- 可替代食材
- 估算用量
- 本地购买渠道和价格信息，如果配置了相关上下文

这样你不会为了做一顿饭买一堆用不完的东西。

### 4. 冰箱与库存管理

CookGPT 可以维护一个简单的厨房库存：

- 记录你买了什么
- 记录你用掉了什么
- 根据当前库存优先推荐合适菜谱
- 做完饭后给出库存更新建议

这让下厨从一次性冲动变成一个可持续的生活系统。

### 5. YouTube 菜谱深度处理

很多好菜谱藏在视频里，但视频不适合边做边看。

CookGPT 可以从可信 YouTube 创作者的视频中提取字幕和元数据，整理出：

- 食材
- 顺序
- 时间线
- 火候
- 操作要点
- 不确定信息标注

默认规则是：只提取字幕和元数据，不下载视频或音频。

### 6. 新手友好的下厨模板

即使你几乎不会做饭，或者只有很基础的厨具，CookGPT 也可以根据你的实际设备生成方案。

比如：

- 只有电饭锅
- 只有一口锅
- 没有烤箱
- 不想洗太多碗
- 只想 20 分钟内吃上饭

它会把复杂菜谱降维成你能执行的版本，而不是给你一份看起来高级但做不出来的菜单。

### 7. 健康目标与忌口适配

如果你有明确目标，可以写进 profile：

- 减脂
- 增肌
- 高蛋白
- 控糖
- 低油
- 少盐
- 过敏原
- 宗教或医疗忌口
- 不喜欢的食材

CookGPT 会在推荐和改写菜谱时优先考虑这些限制。

它不是医疗建议工具，但可以帮助你把日常饮食决策变得更清楚、更稳定。

## 初始化说明

首次使用前，请先创建个人数据目录：

```text
~/.codex/user-data/solo-kitchen/
```

推荐从本仓库的模板复制：

```text
user-data-template/
├── profile.json
├── inventory.json
├── trusted_sources.json
├── obsidian_sources.json
├── obsidian_recipes_index.json
└── recipe_knowledge.jsonl
```

复制后，你至少需要配置下面几项。

### 1. 配置 profile.json

`profile.json` 是你的厨房档案。它决定 CookGPT 如何理解你的饮食需求。

建议填写：

- 所在城市或区域
- 常用买菜渠道
- 默认一餐预算
- 忌口、过敏原、宗教或医疗限制
- 喜欢和不喜欢的口味
- 可接受辣度
- 可用厨具
- 常备调料
- 平时愿意花多少时间做饭
- 可接受的复杂度和清洁成本
- 健康目标，如减脂、增肌、控糖等

默认份量是一人一餐：

```json
{
  "default_serving": {
    "people": 1,
    "meals": 1
  }
}
```

### 2. 配置 inventory.json

`inventory.json` 用来记录你当前有什么食材。

初始化时可以为空：

```json
{
  "version": 1,
  "items": []
}
```

之后你可以逐步告诉 CookGPT：

```text
我买了 6 个鸡蛋、2 个番茄、一袋菠菜。
```

或者：

```text
今晚用了 2 个鸡蛋和 1 个番茄。
```

CookGPT 会根据这些信息更新库存，并在推荐菜谱时优先使用已有食材。

### 3. 配置 trusted_sources.json

`trusted_sources.json` 用来定义你信任的菜谱来源。

你可以加入：

- YouTube 创作者
- 菜谱网站
- 书籍
- 自己的 Obsidian 菜谱笔记
- 其他可信来源

CookGPT 的原则是：推荐必须有来源，不能凭空生成菜谱。

如果某个来源你不信任，也可以放进排除列表。例如：

```json
{
  "name": "example source",
  "url_pattern": "example.com",
  "reason": "不作为可信菜谱来源"
}
```

### 4. 可选：配置 Obsidian 菜谱库

如果你有自己的 Obsidian 菜谱笔记，可以配置：

```json
{
  "vault_path": "",
  "primary_recipe_dir": "",
  "recipe_dirs": []
}
```

配置后，CookGPT 会优先读取你的本地菜谱，再考虑外部来源。

默认规则是只读，不会自动写回 Obsidian，除非你明确要求保存或同步。

### 5. recipe_knowledge.jsonl 是什么

`recipe_knowledge.jsonl` 是本地菜谱知识缓存。

它用来记录已经验证过的菜谱摘要，包括：

- 来源
- URL
- 核验日期
- 菜名
- 原始份量
- 食材
- 摘要做法
- 火候提示
- 适配备注

它不应该存整篇原文，也不应该存 AI 自创菜谱。

## 推荐的首次提问

初始化完成后，可以这样问：

```text
用 solo-kitchen 根据我的库存推荐一顿一人食，给我买菜清单和操作流程图。
```

或者：

```text
我今天只有 30 分钟，想吃高蛋白、少洗锅的一人食，根据库存给我一个方案。
```

或者：

```text

```

## Sample 应用案例

### 场景：一个 Vibe Coder 的 30 分钟晚餐

晚上 8:30，你刚结束一天的 coding。脑子已经被需求、bug、prompt 和产品决策占满，不想再打开外卖软件刷 20 分钟，也不想凭感觉乱做一顿。

你告诉 CookGPT：

```text
我现在冰箱里有鸡蛋、番茄、菠菜、冷冻虾仁和米饭。
今晚想吃一顿一人食，高蛋白，少油，30 分钟内完成，最好少洗锅。
```

CookGPT 会根据你的 profile、库存和 trusted sources，生成一份可执行方案，例如：

- 推荐一道来源明确的菜谱，并说明为什么适合当前库存
- 把原菜谱缩放成一人份
- 列出需要用掉的库存和可选购买项
- 给出备菜顺序：解冻虾仁、切番茄、洗菠菜、打蛋
- 给出最终烹饪时间线：先炒番茄出汁，再下虾仁和蛋，最后处理米饭或汤底
- 标出火候：中火炒番茄、小火收汁、虾仁变色即停
- 生成 Mermaid 操作流程图
- 最后提示库存更新：鸡蛋 -2、番茄 -1、虾仁 -100g、菠菜 -一把

最终你得到的不是一句“做个番茄虾仁蛋饭吧”，而是一份可以直接照着做的厨房执行计划。

这就是 CookGPT 的核心价值：它把“今天吃什么”的决策压力，转化成一个清楚、轻量、可执行的生活流程。

## 设计原则

CookGPT 遵循几条简单原则：

- 不凭空编菜谱
- 优先使用可信来源
- 默认一人一餐
- 先考虑真实库存
- 把复杂菜谱翻译成可执行步骤
- 不默认做周计划、剩菜计划或过期提醒
- 不替代医生、营养师或医疗建议
- 用户数据放在 skill 文件夹之外
